## 3. What worked end-to-end this session

* **School-student login** with username + password (no email needed).
* **Class-link inheritance**: student dashboard immediately shows the school plan badge, the class assignment, and the Bloom mastery panel scoped to this school.
* **Assigned-quiz Start → Begin test → 12-question runner** with timer, per-question chips, navigation, and submit.
* **Result page** shows score, time, strongest/weakest Bloom, radar chart, and per-question pacing.
* **Cross-role write-through**: 1 minute after submit, the teacher's quiz detail flipped CLASS AVERAGE from `—` → **92%** with the student's name and timestamp in the Attempts table.
* **Profile page** renders correctly with school-admin-managed copy and Appearance section.

## 5. Recommendations (additive to the Teacher report)

1. **Fix Bug #25** by surfacing class-quiz attempts on `/student/progress` (or rename the sidebar item to "My Progress" to match the page title and add a separate "Class results" page).
2. **Fix Bug #26** by *not* showing class-assigned quizzes in the My Practice list, or by hiding the delete (🗑) action for class quizzes.
3. **Fix Bug #27** by hiding the per-user "Free / Upgrade" pill when the user is a school student on a school plan.
4. **Fix Bug #24** by labelling per-question pacing rows `Q1` through `QN`.
5. (Inherited) Fix the underlying RSC-cache / RLS bugs from the teacher report — those continue to bite the student dashboard at first load.
