== lib/aiClient.ts (unified AI client) ==

Groq-primary + Gemini-fallback wrapper around groq.ts. Every caller
automatically falls over to Gemini when Groq returns 429 (daily 100k-token
cap exhausted) or 5xx. No call-site changes -- swap
`import { groqJSON } from "@/lib/groq"` for
`import { aiJSON } from "@/lib/aiClient"`.

Behavior:
- Try Groq first (fast, free).
- On 429 (rate-limited) -> fall through to Gemini.
- On 5xx -> fall through to Gemini.
- Other 4xx + parse errors -> rethrow (don't mask logic bugs).

Motivates: 100+ pilot users on AI-heavy features (Teach-Back,
Misconception, Generate, Tutor) can saturate Groq's free tier mid-session.

== docs/RAZORPAY_AUDIT_2026_05_17.md ==

Webhook + checkout security audit of app/api/razorpay/webhook/route.ts.

Verdict: well-implemented for the common case. No CRITICAL findings.
The critical primitives are in place: HMAC-SHA256 with timing-safe
comparison, idempotency via unique partial index on razorpay_payment_id,
amount-mismatch detection, event filtering (payment.captured / order.paid
only), race-safe insert swallowing 23505, school_id nulled on B2C path.

Medium/low items noted (R1: started_at bumped on every renewal UPDATE
distorts (expires_at - started_at) cohort math; R2: notes.user_id
trusted without cross-verification, mitigated by amount-match; etc.).
