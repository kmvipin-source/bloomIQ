#68 — Live preview modal (TestPreviewModal.tsx)
============================================================

A "Preview as student" button in the page header opens a modal showing
the test exactly as a student will see it: stem + 4 lettered options
per question, NO correct answer marked, header with test name + time
limit + question count. Backdrop click and Esc both close. Disabled
until at least one question is selected.

KEY LESSON LEARNED FROM THE PREVIOUS FAILED ATTEMPT:
  A Round-13 attempt at this lived inline in the composer's JSX and
  the modal ended up inserted at the wrong function boundary (it
  landed inside ClassFitCard rather than ComposerInner), with a
  fragment <> opened in ComposerInner but closed in ClassFitCard.
  tsc rejected the resulting structural breakage.

  Round 14 fix: extract the modal into its own component file
  (components/TestPreviewModal.tsx). The composer just renders
  <TestPreviewModal/> from its existing JSX tree — no surgical
  insertion into the tree, no structural risk.

#71 — AI-suggested composition
       (AISuggestComposeModal.tsx + /api/teacher/quiz-suggest)
============================================================

A "Suggest a test" button (purple-emerald gradient pill) in the header
opens a modal where the teacher picks:
  - Class (optional, defaults to the currently-selected class).
  - Topic focus (optional ILIKE filter).
  - Depth: Broad (Remember + Understand-heavy), Mid (Apply + Analyze),
    or Deep (Analyze + Evaluate + Create).
  - Count: 3-40 questions.

The server reads the teacher's approved question bank, filters by
class + topic, then walks a Bloom-mix template for the chosen depth,
preferring topic spread within each Bloom slot. Returns:
  - question_ids        — array to populate selectedIds with
  - bloom_breakdown     — final mix (e.g. {apply: 4, analyze: 4, evaluate: 2})
  - topics              — distinct topics in the suggestion
  - bank_size, matched  — diagnostic counts

The teacher previews the composition in the modal (bank size, Bloom
mix, topics) and clicks "Apply to composer" to load it.

Implementation notes:
  - Pure server-side diversity-aware sampler. NO LLM call. The "AI"
    framing is product-facing; the feature is cost-zero and
    millisecond-fast. Important since this button will be one of the
    most-pressed in the composer.
  - Each Bloom bucket is shuffled so successive suggestions don't
    surface the same 10 questions repeatedly.
  - When the bank doesn't have enough questions to fill the requested
    count, the modal surfaces an amber note pointing the teacher at
    /teacher/generate.

============================================================
#70 — Quick mode toggle
============================================================

A "Quick mode" checkbox in the page header. When on, the right rail
(name / subject / time / marking / save) is dimmed to opacity-50 +
pointer-events-none UNTIL the teacher has picked at least one question
— focusing attention on the bank first. Once a question lands in the
cart, the rail brightens to draw the teacher forward.

Preference is persisted in localStorage so it sticks across sessions.

This is the lighter version of a true 3-screen wizard. If a heavier
wizard is wanted later (Next/Back navigation, fixed step screens),
it can be a follow-up PR.
