QA Audit Round 16: Student-side flows — auth race, free-expired
server gate, focus debounce. 3 of 5 findings fixed; 2 deferred.

After completing the 17/17 Build & Assign sweep in Round 15, started
the next audit module — student-side flows. First-pass findings
documented + the top 3 production-quality wins shipped here.

============================================================
#72 — HIGH — /student/layout auth race (same shape as /teacher)
============================================================
Symptom: first sign-in as a student → land on /student → layout calls
/api/auth/me → transient 401 from Supabase auth-server propagation
delay (~300-800ms) → falls through to user_metadata.role → empty for
school-onboarded students → role-router redirects to /login. Same
exact pattern that Round 8 #53 fixed for /teacher/layout, just on
the student side this time.

Fix: 3-attempt retry loop with 300ms backoff before falling through
to user_metadata. session_superseded responses still short-circuit
the retry and sign out (real "another device signed in" case).

============================================================
#73 — MEDIUM — /api/student/attempt-start lacks server-side
              free-expired gate
============================================================
Symptom: the /student layout client-side redirect catches expired
Free-trial users, but a stale tab / scripted call / racing client can
still POST /api/student/attempt-start. The check_attempt_quota
trigger handles the daily cap, but the time-based / admin-suspended
expiry was unenforced on the server.

Fix: server-side check before insert. When the caller's subscription
is tier=free + is_trial=true + (status in {suspended,cancelled} OR
expires_at < now), return 410 Gone with code "free_expired".

============================================================
#74 — MEDIUM — /student/layout focus listener thrashes on rapid
              tab toggles
============================================================
Symptom: every focus event runs setOk(false) → spinner flash → full
re-fetch of /api/auth/me. Browser notifications + OS focus shuffling
can fire N focus events per second; layout chains N redundant
fetches with N visible spinner flashes between them.

Fix: 5-second debounce on the focus check. The first focus after a
5s quiet window fires normally; subsequent focuses within 5s no-op.

============================================================
