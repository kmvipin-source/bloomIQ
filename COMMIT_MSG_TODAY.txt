#58 — Class-fit topic-coverage gap (CLIENT-SIDE)
============================================================
When a class is selected AND there are picked questions, surface a
new amber "Uncovered class topics" card listing topics that exist in
the teacher's bank for this class but aren't in the current
selection. Click a chip to filter the bank to that topic. Caps at
12 chips with "+N more" overflow.

Pure client-side heuristic — no API change. Reads from the already-
loaded `bank` array filtered by class_id, diffed against the topic
set of `selectedQuestions`. Zero new network calls.

============================================================
#63 — "Show selected only" toggle on the bank
============================================================
New checkbox at the head of the bank filter row. When on (and
selectedIds.length > 0), the bank list view is filtered to only
the currently-selected questions. Layered on top of the existing
`filtered` useMemo via a `displayedBank` derivation — keeps the
filter pipeline clean.

Disabled when selection is empty (with a friendly tooltip).

============================================================
#65 — Recent-topic chip dedup
============================================================
The Recent: chip row was rendering topics that ALSO appeared in
the topicCounts dropdown right below, doubling the same chip in
two visually-similar rows. Now the Recent row dedups against
topicCounts via a Set lookup.

============================================================
#69 — Save composition as template (localStorage)
============================================================
New "Templates" button in the page header opens a dropdown listing
saved filter templates. A template captures the FOUR filter values
(topicFilter / bloomFilter / categoryFilter / search) — not the
question IDs — so re-applying re-runs the same rule against the
LIVE bank each time.

  • Save current → prompts for a name, stores up to 20 templates
                   in localStorage key `zcoriq.composer.templates.v1`.
  • Apply        → restores all four filters.
  • Delete       → removes a single template.

LocalStorage-backed for this round. A future PR can promote
templates to the DB when teachers want them to follow them across
devices.
