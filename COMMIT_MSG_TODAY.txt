ROUND 8 — login race + AI topic disambiguation (2 fixes)
=========================================================

#53 HIGH /teacher/layout.tsx — first-time-login race bounces teachers
        back to /login. F23 follow-up.

  Symptom: User signs in correctly at /login/school (teacher tab). Lands
  on /teacher. Layout calls /api/auth/me. Supabase auth-server eventual-
  consistency returns 401 for ~300-800ms after sign-in. Layout falls
  through to user_metadata.role — which is empty for school-onboarded
  teachers — so role stays "" and the role-router redirects back to
  /login. Teacher logs in again, auth has propagated, all is fine.

  Fix: retry /api/auth/me up to 3 times with 300ms backoff before falling
  back to user_metadata. Eliminates the race in the first attempt. The
  session_superseded branch (real "another device signed in" case) still
  bypasses the retry and signs out immediately, as before.

#54 HIGH /api/generate SYSTEM prompt — short acronym topics under multi-
        subject exam contexts produced wrong-subject questions.

  Symptom (reported): teacher entered topic "LCM" with teaching context
  "JEE" and got Linear/Circular Motion questions. The teacher meant
  Least Common Multiple. AI defaulted to the JEE-physics interpretation
  because the system prompt + exam framing skew toward physics.

  Root cause: the SYSTEM prompt has no acronym-disambiguation rule. The
  teaching context selects JEE-style framing, the topic is short and
  ambiguous, and the AI picks whichever interpretation is most exam-
  associated. The acronym "LCM" maps to a real JEE physics topic
  (Linear/Circular Motion) so the model has no reason to default to math.

  Fix: add explicit rule #7 to the SYSTEM prompt — short acronyms (≤6
  chars) that have multiple plausible meanings in the chosen exam's
  subjects MUST default to the most common primary/secondary education
  meaning, not a specialized exam-jargon interpretation. Examples
  enumerated in the prompt: LCM, HCF, ROI, PI, AC, DC. Model must also
  write a one-line internal INTERPRETATION before generating (helps
  debug + verifier).

ROUND 9 — /teacher/generate visual redesign (color-coded sections)
==================================================================

User feedback after Round 7: the collapsible Advanced wasn't enough —
"some logical grouping/colouring can be done as per modern day app
design". Applied a step-numbered, color-coded section system:

  Step 1 · Which class is this for?         → sky-blue left border
  Step 2 · Who are you teaching today?      → sky-blue left border
  Step 3 · What kind of test are you making? → purple left border (optional)
  Step 4 · Source & question mix             → emerald-green vertical
                                                accent rail (primary area)

The validation banner was also upgraded: bigger padding, rounded-xl,
icon (⚠ / ℹ) on the left, shadow. Catches the eye when blocking
issues fire (e.g. JEE-difficulty + Class-5-8 mismatch).
