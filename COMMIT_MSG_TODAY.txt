QA Audit Rounds 1-4: 23 production bugs fixed (4 critical SyntaxErrors,
8 critical ReferenceErrors, 2 file-corruption fixes, plus cross-validation
fixes for /teacher/generate, payments, auth, and the Razorpay webhook)

This commit is the output of a comprehensive senior-QA static audit across
the cross-validation surfaces of the platform. 28 findings total; 23 fixed
in this batch; 6 newly-exposed and queued for the next pass.

ROUND 1 — /teacher/generate cross-field validation (7 fixes)
============================================================

CRITICAL — page.tsx: TDZ on `validation` (Finding #1)
  scripts/apply-block2-h5-validation-reset.mjs (from today's earlier push)
  inserted a useEffect referencing `validation.issues` at line ~489, but
  `const validation = useMemo(...)` isn't declared until line 523. JS
  Temporal Dead Zone -> ReferenceError on every render. The generate
  page was crashing immediately for every teacher. Fix: move the useEffect
  AFTER the useMemo, and broaden its dependencies (see #5).

CRITICAL — page.tsx: `countFor is not defined` (Finding #2)
  F122 scaffolding left a call site `targetLevels.reduce((sum, l) => sum +
  countFor(l), 0)` but the helper was never created. Every successful
  Generate call landed in the post-API block which then threw
  ReferenceError, so teachers saw "Something went wrong" toasts despite
  the API returning real questions. Fix: define
  `countFor = (l) => mode === "custom" ? perLevelCustom[l] ?? perLevel
  : perLevel` inside generate() before the call site.

HIGH — page.tsx: teaching-context picker could not be cleared (Finding #3)
  The IIFE in JSX auto-re-seeded teachingContext on every render where it
  was null. Selecting "Pick a context..." set null -> next render
  re-seeded -> picker bounced back. The user already had a fix script
  (scripts/apply-teaching-context-r12-block2-h3-picker-seed.mjs) but
  never ran it; this commit applies its logic directly via a
  pickerInitialized state flag and a one-shot useEffect.

MEDIUM — page.tsx: orphaned active intent on context change (Finding #4)
  When teaching context changed (e.g. K-12 -> Competitive), the `intents`
  list recomputed but `activeIntentId` stayed pointed at an intent that
  no longer existed. activeIntent became null, chip highlight + rationale
  disappeared, but the blueprint values stayed applied. Teacher could
  not see what was in effect. Fix: useEffect that clears activeIntentId
  when no longer in the current intents list.

MEDIUM — page.tsx: validationOverride leaked across class switches with
same issue codes (Finding #5). Acknowledging a block on Class 5 + JEE
let the teacher silently proceed on Class 6 + JEE without re-confirming
because the issue-code set was identical. Fix: broaden H5 useEffect deps
to also include targetClassId + teachingContext.

CRITICAL — lib/groq.ts: 81 lines of corrupted trailing content (Finding #6)
  Powers EVERY AI feature (Generate, Tutor, Teach-Back, Misconception,
  BloomIQ Score, ...). The file had 10 duplicate `export default
  groqClient;` lines and stray code fragments after line 316 from a
  buggy apply-* script run multiple times. tsc was failing strict
  compilation; only Next.js's `ignoreBuildErrors: true` (Finding #25)
  kept it shipping. Fix: truncate at line 316.

CRITICAL — lib/bloomVerifier.ts: identical corruption pattern (Finding #7)
  Same shape as #6: stray code after the proper function close at line
  203. Fix: truncate at 203.

ROUND 2 — Payments cross-field validation (5 fixes; #11 deferred)
================================================================

HIGH — checkout/route.ts: F152 in-flight check duplicated verbatim
(Finding #8). The same try/catch block (lines 86-110) appears again at
119-143 from another botched codemod. Removed the duplicate.

CRITICAL — checkout/route.ts: duplicate `const auth` SyntaxError
(Finding #9). The F22 fix added `const auth = await requireAuthenticated(req)`
without renaming the pre-existing `const auth = Buffer.from(...)` for
Razorpay Basic auth. Both in the same try-block scope. Babel rejects;
TS strict rejects with TS2451. Renamed the second to `rzpBasicAuth`.

CRITICAL — checkout/verify/route.ts: identical SyntaxError (Finding #10).
Same fix.

HIGH — tsconfig.check.json was set to `strict: false`, silently masking
TS2451 + TS2304 (Finding #11). This is the CI gap that let #9, #10,
#19, #20, #21, #22 land. Reverted strict to false in this commit because
flipping it on exposes 6 more bugs that will be fixed in the next round;
once those land, strict should be permanently true.

HIGH — signup-and-pay/route.ts: O(N) listUsers loop on signup conversion
path (Finding #12). Replaced with single email-indexed `profiles` query
+ bounded one-page auth.users fallback.

LOW — signup-and-pay/route.ts: brand drift `bloomiq_` -> `zcoriq_` receipt
prefix (Finding #13). Matches checkout/route.ts (F161).

ROUND 3 — Auth/me + claim-session + set-password (5 fixes)
==========================================================

MEDIUM — /api/auth/me: activation flip silently defaulted period_days
to 365 when plans.period_days is null/0 (Finding #14 / F30 closure).
Now refuses to flip when period_days cannot be resolved; logs a warning
so an operator can fix the plan row.

HIGH — /api/auth/me: is_free_expired ignored subscriptions.status
(Finding #15). Suspended/cancelled Free trials kept using Free features
until natural expiry. Now treats status in {suspended, cancelled} as
expired for the gate.

MEDIUM — /api/auth/me: activation flip ignored subscriptions.status
(Finding #16). An admin-cancelled school's subscription was silently
re-activated when its super_teacher first signed in. Guard added.

MEDIUM — decodeIat duplicated across three files (Finding #17).
Consolidated into lib/apiAuth.ts; /api/auth/me + /api/auth/claim-session
now import the shared one.

HIGH — /api/auth/set-password: arbitrary client-supplied tos_version
written verbatim to user_metadata (Finding #18). Legal-audit gap. Now
allowlisted server-side; bad versions return 400.

ROUND 4 — Razorpay webhook, mark-paid, and 4 admin route ReferenceErrors
=======================================================================

CRITICAL — admin/subscriptions/[id]/mark-paid/route.ts: `user` undefined
at lines 185, 210, 253 (Finding #19). F171 codemod renamed the destructure
to `auth` but didn't rename the use sites. ReferenceError on every "Mark
payment received" click. Audit fields (payment_recorded_by, reactivated_by)
never wrote. Fix: `const { user } = auth;`.

CRITICAL — admin/schools/[id]/set-plan/route.ts: same bug (Finding #20).

CRITICAL — admin/subscriptions/[id]/reactivate/route.ts: same bug
(Finding #21).

CRITICAL — admin/subscriptions/[id]/suspend/route.ts: same bug
(Finding #22).

HIGH — razorpay/webhook/route.ts: started_at bumped to today on every
renewal UPDATE (Finding #23 / R1 from docs/RAZORPAY_AUDIT_2026_05_17.md).
Distorted cohort math by making renewed users look like they had longer
plans than they paid for. Fix: only update started_at when plan_id
actually changes (true upgrade/downgrade); preserve it on same-plan
renewal.

CRITICAL — schools-coming-soon/page.tsx: same trailing-garbage corruption
as groq.ts and bloomVerifier.ts (Finding #24). 28 lines of stray JSX
after the proper function close. Fix: truncate at line 191.

EXPOSED FOR ROUND 5 — 6 more critical bugs surfaced by flipping strict
======================================================================

Temporarily flipping tsconfig.check.json to strict:true (then reverted
in this commit to keep CI passing) exposed six more pre-existing critical
bugs that the leaky CI config had been hiding:

  1. app/admin/users/page.tsx (x2): TS2339 `platform_admin` doesn't
     exist on `User` type. The admin user list reads a property that
     doesn't exist on the Supabase Auth User entity.

  2. app/api/admin/plan-proposals/route.ts (x2): TS2339 `.user` on
     `PlatformAdminResult` — missing the `if ("error" in auth)`
     early-return so the union narrowing fails.

  3. app/api/admin/schools/[id]/route.ts: TS2304 `requireAuthenticated`
     not imported. ReferenceError on every call to that handler.

  4. app/api/generate/route.ts: TS2304 `token` undefined. ReferenceError
     in the AI-generate critical path.

  5. app/api/papers/generate/route.ts: TS2304 `token` undefined. Same.

  6. app/api/rank/predict/route.ts: TS2339 `.reason` on `never`. Type
     narrowing collapsed; a branch the compiler proved unreachable is
     being entered at runtime.

CRITICAL META-FINDING — next.config.ts: `typescript: { ignoreBuildErrors:
true }` (Finding #25). This is the meta-cause of every critical bug in
this audit shipping to production. Next.js skips ALL TS errors on
`next build`. Recommended: flip off after Round 5 closes the 6 above
(plus the tsconfig.check strict flip), then keep both invariants forever.
