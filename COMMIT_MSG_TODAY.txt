ROUND 5 — exposed by flipping tsconfig.check.json strict to true (10 fixes)
============================================================================

CRITICAL — admin/users/page.tsx: User type was missing platform_admin
field (Finding #26). The filter chip at line 196 + count at line 270
read u.platform_admin but it wasn't on the type. Strict TS caught it
once the CI gap closed.

CRITICAL — admin/plan-proposals/route.ts: legacy `"err" in auth` /
`auth.err` narrowing across two handlers (Finding #27). The local
`requireAdmin = requirePlatformAdmin` alias was kept but the call sites
weren't updated to the new {error} shape. Both branches now use the
modern narrowing.

CRITICAL — admin/schools/[id]/route.ts: requireAuthenticated used by
DELETE handler but not in the import line (Finding #28). ReferenceError
on every school-delete. Now imported.

CRITICAL — api/generate/route.ts: token used inside genFor() arrow but
not destructured from auth (Finding #29). ReferenceError once the
misconception-seed branch fired. Now destructured.

CRITICAL — api/papers/generate/route.ts: identical pattern (Finding #30).

CRITICAL — api/rank/predict/route.ts: dead-code else branch reading
.reason on a `never`-narrowed eligibility (Finding #31). The isExamMock
guard at line 230-232 narrows verdict to {matches_known_exam,
competitive_exam_other}, both handled by the if + else-if above. The
else is unreachable; replaced its bad property access with a static
safety-net string.

CRITICAL — api/student/adaptive-practice/route.ts: same token pattern as
#29/#30 (Finding #32).

CRITICAL — api/student/quick-test/route.ts: same token pattern (Finding
#33).

MEDIUM — teacher/generate/page.tsx countFor type regression introduced
by my own Round-1 fix (Finding #35): under strict TS, BLOOM_LEVELS
widens to readonly string[] which clashes with countFor's BloomLevel
parameter. Fixed by slicing + casting before reduce.

Plus the meta-invariant closure:

HIGH — tsconfig.check.json: strict flipped to TRUE PERMANENTLY (Finding
#11 closure). The CI invariant is now in place: any future PR that
introduces a TS2304 (Cannot find name), TS2451 (block-scoped redecl),
TS2339 (missing property), or TS2448 (used-before-declared) error will
fail `tsc --noEmit -p tsconfig.check.json`. The full sweep of bug
shapes that landed in production over the past months — duplicate-auth
SyntaxErrors, undefined-user ReferenceErrors, undefined-token
ReferenceErrors, trailing-corruption from runaway codemods — was
caused by this single check config being lax.
