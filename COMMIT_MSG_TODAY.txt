QA Audit Round 7: /teacher/generate cross-validation + visual cleanup (2 fixes)

Two issues caught from a fresh look at the live form (teacher-reported screenshot):

FINDING #45 — HIGH — Question count display mismatch
====================================================
The form had TWO running totals that used different formulas:

  Upper "Total: 6 questions"     (near the per-level inputs)
    -> perLevel * pickedLevels.length          (IGNORED overrides)
  Lower "Will generate 3 questions"  (pre-flight, near the Generate button)
    -> pickedLevels.reduce((s, lv) => s + (perLevelCustom[lv] ?? perLevel), 0)
                                                (HONORED overrides)

Repro: pick 3 Bloom levels, default perLevel=2, set per-level overrides to
1 each. Upper says "6", lower says "3". Teacher loses trust in both.

Fix: single source of truth. Both numbers now use the same override-aware
sum. The upper Total also shows a small "· per-level overrides applied"
badge when overrides are in effect, so the teacher knows why the number
differs from default × levels.

FINDING #46 — HIGH (UX) — Form was visually cluttered
======================================================
On the same screenshot the teacher had to scroll past:
  - Bloom levels picker
  - Customize-per-level inputs (3 number boxes)
  - Reading-level chips (Beginner / Practitioner / Expert)
  - Sub-topic chips (Prime Factorization / GCD / LCM / ...)
  - More-instructions textarea
  - Default questions per level input
  - Numerical % slider
  - Validation banner
  - Generate button

Eight controls in one column, no visual hierarchy. First-time teachers
got lost; the secondary form drowned the primary controls.

Two changes:

  (a) Label dedup. The phrase "Questions per level" appeared twice (one
      header for the per-level override inputs, one header for the
      default-perLevel input). Both with the same wording. Renamed:
        per-level overrides  -> "Customize per level (optional)"
                                 + sub-text "Override the default of N
                                    for specific levels — leave blank
                                    to keep the default"
        default per level    -> "Default questions per level"
                                 (already renamed in Finding #45 fix)

  (b) Collapsible Advanced section. Wrapped GenerateContextChips
      (audience-level + sub-topic + more-instructions) and the
      Numerical % slider inside a <details> element with summary
      "Advanced — fine-tune the mix (audience, sub-topics, numerical %)".
      Closed by default so the primary form (source → topic → class →
      Bloom → Generate) reads top-to-bottom with no distractions.
      Teachers who need the dials click to expand. Down-chevron rotates
      via group-open:rotate-90 for an obvious affordance.
